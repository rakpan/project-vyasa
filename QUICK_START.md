# Quick Start Guide - Project Vyasa

> **The fastest path to running Project Vyasa on DGX Spark (GB10)**

## TL;DR

```bash
# 1. Preflight check
./scripts/preflight_check.sh

# 2. Configure environment
cd deploy && cp .env.example .env && # edit .env

# 3. Start system (first time)
./scripts/run_stack.sh start  # add --opik to include Opik services

# 4. Access console
open http://localhost:3000
```

## Detailed Steps

### 1. Preflight Check (2 minutes)

Validates your DGX Spark environment:

```bash
./scripts/preflight_check.sh
```

**Must pass**: GPU detection, memory (120GB+), ports available.

### 2. Bootstrap Secrets (First Time Only)

```bash
./scripts/init_vyasa.sh --bootstrap-secrets
```

This generates secure credentials in `deploy/.secrets.env` (git-ignored). Credentials are auto-generated by default to prevent hardcoded leaks.

### 3. Configure Environment (5 minutes)

```bash
cd deploy
cp .env.example .env
# Edit .env with your:
# - Model paths
# - GPU IDs
# - HF_TOKEN (HuggingFace Hub token for model downloads)
# Note: Passwords are auto-generated in .secrets.env
```

### 4. Start System (10-30 minutes first time)

Use the unified stack runner (adds `--opik` to include Opik services):
```bash
./scripts/run_stack.sh start
```

### 5. Access Console

Open: **http://localhost:3000**

Login with `CONSOLE_PASSWORD` from `.env`.

### 7. Upload PDF

1. In project workbench, upload PDF to **Seed Corpus**
2. System automatically processes it
3. View extraction results in **Processing** panel

## Common Issues

**Services won't start**:
- Check GPU: `nvidia-smi`
- Check ports: `./scripts/preflight_check.sh`
- Check logs: `./scripts/run_stack.sh logs [--opik] [service]` or `cd deploy && docker compose logs <service>`
- **Port conflicts**: The script automatically detects port conflicts (e.g., port 11435 for drafter). If detected, you'll see resolution options
- **Network issues**: The script automatically creates the `vyasa-net` network when using `--opik`

**Orchestrator fails**:
- Ensure ArangoDB is healthy: `curl http://localhost:8529/_api/version`
- Check logs: `./scripts/run_stack.sh logs orchestrator` or `docker compose logs deploy-orchestrator-1`
- **"uvicorn not found" error**: This is automatically fixed - the orchestrator installs dependencies on startup. If it persists, check that `requirements.txt` is accessible
- **Port conflicts**: The script checks for port conflicts. If port 8000 is in use, change `PORT_ORCHESTRATOR` in `.env`

**Console won't load**:
- Ensure Orchestrator is healthy: `curl http://localhost:8000/health`
- Check logs: `docker compose logs vyasa-console`
